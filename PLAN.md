# خطة بناء نظام «القرآن الكريم» (WPF / .NET 9)

## 1. التقنيات المعتمدة

| العنصر | الاختيار |
|--------|----------|
| الإطار | WPF على .NET 9 |
| نمط التصميم | MVVM (CommunityToolkit.Mvvm) |
| قاعدة البيانات | SQLite (ملف محلي، لا يحتاج إنترنت) |
| الوصول للبيانات | Microsoft.Data.Sqlite / EF Core |
| واجهة المستخدم | RTL كامل + خطوط قرآنية مضمّنة |

> ملاحظة: نخزّن نص القرآن وبيانات الجذور والإعراب محلياً في SQLite حتى يعمل
> التطبيق دون اتصال بالإنترنت ويكون البحث سريعاً.

## 2. مصادر البيانات المطلوبة

1. **نص القرآن** (سور، آيات، أرقام الصفحات) — من مصدر مفتوح موثوق
   (مثل Tanzil / QUL).
2. **الجذور الصرفية** لكل كلمة — لدعم البحث بالجذر.
3. **بيانات الإعراب** — اختيارية في المرحلة الأولى (يمكن تأجيلها).
4. **الخطوط القرآنية**: العثماني، الثلث، الرقعة (ملفات `.ttf`).

## 3. بنية المشروع (Projects)

```
QuranAlKareem.sln
├── QuranAlKareem.App        (WPF — الواجهة وViewModels)
├── QuranAlKareem.Core       (النماذج، الخدمات، منطق البحث)
└── QuranAlKareem.Data       (قاعدة البيانات والوصول إليها)
```

## 4. مراحل التنفيذ

### المرحلة 0 — التهيئة
- [ ] إنشاء الـ Solution والمشاريع الثلاثة.
- [ ] إضافة الحزم (MVVM Toolkit، SQLite).
- [ ] إعداد RTL والثيم الأساسي.

### المرحلة 1 — عرض المصحف (الميزة 1)
- [ ] تحميل نص القرآن إلى SQLite.
- [ ] شاشة عرض السورة/الصفحة.
- [ ] تبديل الخطوط (عثماني / ثلث / رقعة).

### المرحلة 2 — البحث النصّي (الميزتان 2 و4)
- [ ] حقل بحث + خدمة بحث نصّي.
- [ ] عرض النتائج مع اسم السورة ورقم الآية.

### المرحلة 3 — البحث بالجذر (الميزة 3)
- [ ] ربط جدول الجذور.
- [ ] إرجاع كل المشتقّات والآيات الحاوية لها.

### المرحلة 4 — نسخ الآية والانتقال إليها (الميزة 5)
- [ ] زر نسخ نص الآية.
- [ ] النقر على نتيجة البحث يفتح صفحة المصحف عند الآية.

### المرحلة 5 — الإعراب (الميزة 6)
- [ ] عرض إعراب الكلمة/الجذر المختار.

### المرحلة 6 — اللمسات الاحترافية
- [ ] أنيميشن تقليب الصفحات.
- [ ] تحسين الثيم والتفاصيل البصرية.

## 5. نقطة الانطلاق المقترحة

نبدأ بالمرحلة 0 + المرحلة 1 (عرض المصحف الأساسي مع تبديل الخطوط)،
ثم نتدرّج في باقي المراحل.
